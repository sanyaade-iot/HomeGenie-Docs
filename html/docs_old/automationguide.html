Automation Programs

Introduction

Automation Programs are a powerful feature of HomeGenie that let you customize and automate every aspect of your HA system.
In HomeGenie there are two types of automation programs: Wizard and CSharp.

Wizard Scripts

A Wizard Script do not require any programming knowledge. Every commands sequence can be created with a few clicks/taps on the screen.

All we have to do is to choose one or more conditions and a determine what instructions are to be executed when conditions are met.



The program in the above picture, will turn light A8 on and off twice, every exact hour (current time minute is 0).

When adding a new condition or a new instruction statement, a popup window will first ask to choose the context (aka domain) of the statement.
The context can be one of the following:

  • HomeGenie
    for using automation properties and methods implemented in HomeGenie Automation API
  • X10
    for operating on X10 devices
  • ZWave
    for operating on ZWave devices

After choosing the context, the wizard will list other available options for the selected value.

CSharp Programs

A CSharp program use the power of C# programming language and HomeGenie functions to create programs that can handle very complex automation tasks.
CSharp programs can also communicate with the outside world (eg. gathering weather and traffic infos from internet web services, communicate with other connected devices, send email).



As seen for Wizard programs, C# programs consist of two blocks of code. The Condition Trigger code and the Program code to run. The condition trigger is a block of c# code that must return true or false. If returning true, then the program code is run.

Example Condition Trigger code
// A "return true;" statement at any point of this code block, will cause the program to run.
// For manually activated program, just leave a "return false" statement here.
return Modules.WithName("HSM100 Sensor").MotionDetected;
Example Program Code to run
Modules.InGroup("Living Room").OfDeviceType("Light,Dimmer").On();
Pause(15.0);
Modules.InGroup("Living Room").OfDeviceType("Light,Dimmer").Off();

The program above will turn on for 15 seconds and then off, all lights in the living room whenever a motion is detected from sensor with name "HSM100 Sensor".

A C# program also take advantage of some helper classes that are part of HomeGenie automation api and that make easier handling common automation tasks.

Automation API - Helper Classes

Modules Helper Class

This class is used for managing automation modules. It has methods and properties for selecting and controlling modules.

Selecting modules

Selector Method

Usage Example

WithName( "<comma_separated_names>" )
selects modules by names

// turn ALL modules with name "Ceiling Light" on Modules.WithName("Ceiling Light").On();

WithAddress( "<comma_separated_addresses>" )
selects modules by addresses

// turn off modules with address A4 and A7 Modules.WithAddress("A4,A7").Off();

OfDeviceType( "<comma_separated_types>" )
selects modules by device types
values: "Light", "Dimmer", "Switch", "Sensor"

// dim all modules of type "dimmer" or "light" to 50% Modules.OfDeviceType("Light,Dimmer").SetLevel("50");

InGroup( "<comma_separated_groups>" )
selects modules by groups

// set all dimmers in "Living Room" group to 30% Modules .InGroup("Living Room") .SetLevel("30");

InDomain( "<comma_separated_domains>" )
selects modules by domains

// turn off all X10 lights Modules .InDomain("HomeAutomation.X10") .OfDeviceType("Light").Off();

Controlling modules

Command Method

Usage Example

On()
turn on selected modules

// turn ALL modules with name "Ceiling Light" on Modules.WithName("Ceiling Light").On();

Off()
turn off selected modules

// turn module with address "A4" off Modules.WithAddress("A4").Off();

Toggle()
toggles selected modules

// toggles all modules in the group "Study" Modules.InGroup("Study").OfDeviceType("Light").Toggle();

Querying modules properties

For querying a value from modules, following properties are available. Note that when more than one modules is selected, a property may returns the average value.

Property

Description

Level

double value indicating average % level of selected modules.

IsOn

[readonly] bool value indicating if at least one of selected modules is on.

IsOff

[readonly] bool value indicating if all selected modules are off.

Alarmed

[readonly] bool value indicating if at least one of selected modules is alarmed.

MotionDetected

[readonly] bool value indicating if at least one of selected modules detected motion.

Teperature

[readonly] double value indicating average temperature reported by sensors

Luminance

[readonly] double value indicating average luminance reported by sensors

Humidity

[readonly] double value indicating average humidity reported by sensors

RfRemoteData

[readonly] string value containing raw data as received from RF remote

Examples

if (Modules.InGroup("Bedroom").Temperature < 18 && Modules.InGroup("Bedroom").WithName("Heating Switch").IsOff)
{
    // turn on some heating here =)
    Modules.InGroup("Bedroom").WithName("Heating Switch").On();
}
else if (if (Modules.InGroup("Bedroom").Temperature > 22 && Modules.InGroup("Bedroom").WithName("Heating Switch").IsOn)
{
    Modules.InGroup("Bedroom").WithName("Heating Switch").Off();
}
Automation API - Direct Access

The following commands give low level access to HomeGenie Automation API. These API are based on MIG SDK and allow controlling connected devices and services by using a common protocol interface (that is not discussed in this document).

Method

Usage Example

Parameter( "<mig_parameter_string_value>" )
get the value of a given module parameter

if (Modules.Parameter("Sensor.Alarm") != "0") { ... }

Command( "<mig_command_string_value>" )
set a command to be executed on selected modules

// brighten all X10 lights Modules .InDomain("HomeAutomation.X10") .OfDeviceType("Dimmer").Command("Control.Bright").Execute();

Execute / Set ( ["<optional_parameters_string>"] )
execute selected command

see preceding example


Show MIG string values for Parameter and Command methods

The Parameter method returns the value (string) for a given module parameter (eg. "Status.Level" or "Sensor.Temperature"). If more than one module is selected, it will return only the value of the first selected element.
The Command method executes a given command (eg. "Control.On") on selected modules.

Security Alarm Program

Condition Trigger
// if X10 SC9000 security console is Armed Away
// then set the automation state variable "Security.Armed" to ON
//
if (AutomationStates.WithName("Security.Armed").IsOff && Modules.WithAddress("F12").IsOn) // SC9000 Armed Away
{
    // SC9000 has been armed
    AutomationStates.WithName("Security.Armed").On();
    // activate flashing light of strobe sirens named "Strobe Alarm"
    Modules.WithName("Strobe Alarm").Level = 15;  
}
else if (AutomationStates.WithName("Security.Armed").IsOn && Modules.WithAddress("F12").IsOff) // SC9000 Disarmed
{
    // SC9000 has been disarmed
    AutomationStates.WithName("Security.Armed").Off();
    // turn off all strobe sirens named "Strobe Alarm"
    Modules.WithName("Strobe Alarm").Level = 0;
}
//
// if automation state variable "Security.Armed" is ON,
// then we check for sensors status 
// that can trigger alarms and sirens.
// in this example all motion sensors with the name "HSM100 Sensor"
//
if (AutomationStates.WithName("Security.Armed").IsOn && Modules.WithName("HSM100 Sensor").MotionDetected)
{
    // returning true will make programengine execute 
    // the code for sending mail alert
    // and activating the strobe siren
	return true;	
}
//
return false;
Program Code
if (Modules.WithName("Strobe Alarm").Level != 100)
{
    // turn on all strobe sirens named "Strobe Alarm"
 	Modules.WithName("Strobe Alarm").Level = 100;
    // send email
    string smtpserver = "smtp.myserver.com";    
    Net.MailService(smtpserver).WithCredentials("myuser", "mypass")
        .SendMessage("me@hg.net",           //from
        "mario@rossi.it;pippo@pluto.com",   // recipients
        "Warning",                          // subject
        "Security Alarm Triggered");        // message text
}
