EventsHelper Class

Class instance accessor: When

.ModuleParameterIsChanging

This event handler is used to process module events. When a module parameter changes, module and parameter reference are passed to the "fn" delegate. The delegate "fn" must return a bool value to indicate wether this event has to be passed to other listeners or not.
This event is called just before .ModuleParameterChange event.

.ModuleParameterIsChanging (  fn(module, parameter)  )Returns: EventsHelper

module
Type: ModuleHelper
Reference to the module object that raised the event. Passed as argument of the fn delegate.
parameter
Type: ModuleParameter
Name of the module parameter that is getting a new value. Passed as argument of the fn delegate.
Example snippet:
		When.ModuleParameterIsChanging( (module, parameter) => 
		{
			if (module.Is("Kitchen Motion Sensor") && parameter.Is("Status.Level"))
			{
				// ...
				return false;
			} 
			return true;
		});

.WebServiceCallReceived

It is used to add and handle custom WebService functionalities.

.WebServiceCallReceived (  apicall,  fn( args )  )Returns: EventsHelper

apicall
Type: string
A string representing the API call prefix we want to handle.
args
Type: string
A string passed as argument of the fn delegate and that contains the url part coming after the apicall prefix.
The fn delegate must return a string containing the response sent to the client.
For instance, if we have an "Hello World" program that respond to the custom API call:

    http://<hg_server_address>/api/Hello.World/Greet

then, "Hello.World" will be our apicall while "Greet" will be the args string passed to the fn delegate.

Example snippet:
		When.WebServiceCallReceived( "Hello.World", (args) => 
		{
			var returnstring = "";
			if (args == "Greet")
			{
				returnstring = "Hello HomeGenie World!";
			} 
			return returnstring;
		});
